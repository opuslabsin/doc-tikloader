Tikloader Documentation

These instructions will help you in setup mobile application.

Introduction

Tikloader is a TikTok videos download manager.

Setup

  1. Android

    Basic requirements

    • GIT Setup https://git-scm.com/downloads
    • Java Development Kit https://www.oracle.com/java/technologies/javase-jdk8-downloads.html
    • Android Studio https://developer.android.com/studio
    • Firebase Account https://console.firebase.google.com
    • SHA-1 fingerprint of your machine https://cloud.google.com/solutions/mobile/mobile-firebase-app-engine-flexible#generating_a_sha-1_fingerprint_for_the_app
    • SHA-1 fingerprint of your release key for release build https://stackoverflow.com/questions/3997748/how-can-i-create-a-keystore

    Firebase Setup

    • Create a Firebase account or log into an existing account.

    • Click Add project. Follow the remaining setup steps and click Create project.

    • After the wizard provisions your project, click Continue.

    • In the Overview page of your project, click the Settings gear and then click Project settings.

    • Click Add Firebase to your Android app and follow the wizard

    • In Debug signing certificate SHA-1, enter the SHA-1 value you generated in the previous section.

    • Click Register app.

    • Download the google-services.json file and place it in app folder.

    • Click Next in the Download config file section and follow the rest of the steps if required.

    Project Setup

    • Open android studio.
    • Click on Open an existing project and Select the source code.
    • Open app level build.gradle to edit with your values.

    Values you’ll be needing:

    • applicationId
    • app_name
    • ad_unit_id

    Refactoring

    • Logo and placeholders used in app are png images which you will find in app/src/main/res/drawable folder.
    • To change the images in app simply replace the images at above mentioned location with your images.
    • To change application launcher icon change ic_launcher.png in
      • mipmap-hdpi
      • mipmap-mdpi
      • mipmap-xhdpi
      • mipmap-xxhdpi
      • mipmap-xxxhdpi
    • To change application colors, change values in app/src/main/res/values/colors.xml
    • To change application strings, change values in app/src/main/res/values/strings.xml
    • Currently application supports 6 Languages
      • English
      • Arabic
      • Spanish
      • French
      • Indonesian
      • Portuguese

    Configuration

    • Open android studio.
    • Click on Open an existing project and Select the source code.
    • Open app level build.gradle to edit with your values.
    • Open app/build.gradle, you need to edit/add values in respective fields in defaultConfig under android
    • Fill in applicationId with your package name which you registered on firebase console
    • Fill in app_name with your desired app name

    AdMob setup

    • Open android studio.
    • Click on Open an existing project and Select the source code.
    • Open app level build.gradle to edit with your values.
    • Make sure ENABLE_ADMOB is set to true if you desire to setup AdMob in your application.
    • If not, set it to “false” to disable AdMob
    • If ENABLE_ADMOB is set to true, mention your banner add unit id in ad_unit_id
    • Open app/src/main/AndroidManifest.xml
    • Mention your AdMob application id in com.google.android.gms.ads.APPLICATION_ID’s value in AndroidManifst.xml file.

    Build App

    • Open android studio.
    • Click on Open an existing project and Select the source code.
    • In the toolbar, select your app from the run configurations drop-down menu.
    • From the target device drop-down menu, select the device that you want to run your app on.
    • Click run.
    • For more info https://developer.android.com/studio/run

    Generate Keystore

    • In the menu bar, click Build > Build > Generate Signed Bundle/APK.
    • In the Generate Signed Bundle or APK dialog, select Android App Bundle or APK and click Next.
    • Below the field for Key store path, click Create new.
    • On the New Key Store window, provide the following information for your keystore and key
    • Key store path: Select the location where your keystore should be created.
    • Password: Create and confirm a secure password for your keystore.
    • Alias: Enter an identifying name for your key.
    • Password: Create and confirm a secure password for your key. This should be different from the password you chose for your keystore.
    • Validity (years): Set the length of time in years that your key will be valid. Your key should be valid for at least 25 years, so you can sign app updates with the same key through the lifespan of your app.
    • Certificate: Enter some information about yourself for your certificate. This information is not displayed in your app, but is included in your certificate as part of the APK.
    • Once you complete the form, click OK.

    Build Signed Apk

    • Build > Generate Signed Bundle/APK.
    • In the Generate Signed Bundle or APK dialog, select either Android App Bundle or APK and click Next.
    • Select a module from the drop down.
    • Specify the path to your keystore, the alias for your key, and enter the passwords for both.
    • If you are building and signing an APK, you need to select which Signature Versions you want your app to support.
    • Click Finish.
